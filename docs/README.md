# Git Commit Message Template — Documentation

Map of the project's documentation. The main usage guide lives in the root
[`README.md`](../README.md); this folder gathers the supporting material.

| Document                                   | Question it answers      | When to read it          |
| ------------------------------------------ | ------------------------ | ------------------------ |
| [`product/roadmap.md`](product/roadmap.md) | Where is it going?       | To learn the priorities  |
| [`decisions/`](decisions/README.md)        | Why each decision?       | Before re-debating it    |

## How to maintain this documentation

- Update the **"Last updated"** line when you edit a document.
- Record relevant decisions as [ADRs](decisions/README.md).
- Keep this index up to date if you add or remove documents.
